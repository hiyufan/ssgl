## 1. 产品概述

### 1.1 产品背景

当前竞赛管理存在以下痛点：
- 审批流程依赖线下或简单表单，效率低下且缺乏追溯性
- 赛事预计划审核完全依赖人工，耗时且标准不统一
- 学生对教师辅导质量缺乏反馈渠道
- 获奖名单核定与结算工作繁琐易错
- 往届优秀项目经验未能有效沉淀和复用
- 团队组建时跨学科资源整合困难

### 1.2 产品定位

构建一个 **AI 驱动的竞赛全流程管理平台**，覆盖赛事从计划、执行到结算的完整生命周期，并通过 AI 能力提供智能化辅助。

### 1.3 产品目标

| 目标 | 衡量指标 |
|------|----------|
| 提升审批效率 | 审批周期缩短 50% |
| 标准化审核 | AI 审核覆盖率达 80% |
| 改善教学质量 | 学生评价参与率达 90% |
| 数据驱动决策 | 统计报表覆盖所有核心业务 |
| 智能辅助 | 6 个 AI 工具全部上线可用 |

### 1.4 目标用户

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| 学生 (Student) | 竞赛参赛者，团队成员 | 便捷报名、获取 AI 辅助、评价教师 |
| 教师 (Teacher) | 竞赛指导教师，审批人 | 高效审批、查看团队表现、获得学生反馈 |
| 管理员 (Admin) | 赛事组织者，系统管理者 | 全局管控、数据分析、赛事结算 |

### 2.1 用户管理模块

#### 2.1.1 用户注册与登录

**功能描述：** 支持用户注册、登录、登出，基于 JWT 的身份认证。

**需求详情：**

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| UM-001 | 用户注册 | P0 | 支持用户名、邮箱、手机号注册，需选择角色 |

**验收标准：**
- [ ] 注册时邮箱唯一性校验
- [ ] 密码强度要求：至少 8 位，包含大小写字母和数字
- [ ] JWT Token 有效期 2 小时，Refresh Token 有效期 7 天
- [ ] 登录失败 5 次后锁定账号 15 分钟

- [ ] 审批驳回后学生可修改重新提交

- 作为学生，我需要在辅导结束后评价教师，提供真实反馈帮助改进

**验收标准：**
- [ ] 每个学生对每个教师同一竞赛只能评价一次
- [ ] 评价提交后不可修改
- [ ] 教师只能看到汇总数据，不能看到单个学生的评价（匿名保护）
- [ ] 汇总数据包括各维度平均分、评价数量、文字反馈列表

## 4. 技术架构

### 4.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | React 18 + JSX + Babel | 已完成，飞书风格 UI 组件库 |
| UI 组件 | 自定义组件 (Btn, Card, Table, Modal, Badge, Toast 等) | 飞书风格，已在 `frontend/platform/ui.jsx` 实现 |
| 状态管理 | React Context + useState | 轻量级，适合当前规模 |
| 图表 | 自定义 SVG 图表 (BarChart, LineChart, DonutChart) | 已在 `frontend/platform/stats.jsx` 实现 |
| 后端 | Go + Gin/Fiber | 高性能，强类型 |
| AI 服务 | Python + FastAPI | AI/ML 生态最佳 |
| RAG | LangChain / LlamaIndex + pgvector | 成熟 RAG 框架 |
| 数据库 | PostgreSQL + pgvector | 关系型 + 向量一体化 |
| 缓存 | Redis | 高速缓存，频率限制 |
| 认证 | JWT + bcrypt | 无状态认证 |
| 部署 | Docker Compose | 简单本地开发 |

## 6. 风险与缓解

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| AI 审核质量不稳定 | 高 | 中 | RAG + 往届项目库 + Prompt 工程优化 |
| 功能范围过大 | 高 | 高 | 严格阶段门控，每阶段必须完成再进入下一阶段 |
| 单人开发疲劳 | 中 | 中 | AI 编程工具 + 清晰任务分解 |
| LLM API 成本 | 中 | 中 | 本地模型兜底 + 缓存策略 |
| Demo 压力 | 中 | 高 | 预置演示数据 + 每个功能备份方案 |
| 集成复杂度 | 中 | 中 | API 优先设计，Go ↔ Python 通过 REST 通信 |
